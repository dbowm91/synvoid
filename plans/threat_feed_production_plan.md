# Implementation Plan: Threat Feed Production & CLI Exporter

**Status**: Proposed
**Last Updated**: 2026-04-28

---

## Objective

Implement the tools necessary for Global Nodes to produce and sign authoritative threat intelligence feeds. This includes adding CLI commands for exporting signed JSON feeds and implementing the signing logic in the core mesh library.

## Background

MaluWAF V2 introduced the `ThreatFeedClient` (consumer). To support the open-source community and multi-cluster deployments, we need a "Producer" side that can curate mesh observations and publish them as signed "Gold Standard" feeds.

---

## Phased Implementation Plan

### Wave 1: Core Signing & Serialization
Implement the deterministic hashing and signing logic in the mesh library.

| Task ID | Component | Description | Implementation Details |
| :--- | :--- | :--- | :--- |
| **P1.1** | `mesh/threat_intel.rs` | Deterministic Hashing | Implement `ThreatIntelligenceManager::get_feed_signable_content(indicators: &[ThreatIndicator]) -> String`. |
| **P1.2** | `mesh/threat_intel.rs` | Payload Generation | Implement `ThreatIntelligenceManager::create_signed_feed(site_id: Option<&str>, key: &VerifyingKey) -> ThreatFeedPayload`. |
| **P1.3** | `waf/threat_intel/` | Export Logic | Ensure `ThreatFeedPayload` and `ThreatFeedIndicator` have `serde` Serialize/Deserialize implemented correctly for JSON. |

### Wave 2: CLI Command Implementation
Add the `--export-threat-feed` command to the main MaluWAF binary.

| Task ID | Component | Description | Implementation Details |
| :--- | :--- | :--- | :--- |
| **P2.1** | `src/main.rs` | CLI Argument Parsing | Add `export_threat_feed: bool`, `sign_with: Option<PathBuf>`, and `site_id: Option<String>` to the `Args` struct. |
| **P2.2** | `src/master/commands.rs` | Export Handler | Implement `handle_export_threat_feed(args: &Args)`. This must load the local threat store, filter by site, sign the payload, and write to stdout or a file. |
| **P2.3** | `src/master/commands.rs` | Key Loading | Add support for loading Ed25519 private keys from files or using the configured node key. |

### Wave 3: Integration & Documentation
Ensure the exporter works seamlessly with the existing consumer.

| Task ID | Component | Description | Implementation Details |
| :--- | :--- | :--- | :--- |
| **P3.1** | Tests | Round-trip Verification | Add a test that exports a feed using the CLI logic and then parses/verifies it using `ThreatFeedClient`. |
| **P3.2** | Documentation | Update README/Docs | Add instructions for running an "Authoritative Intel Node" to `docs/THREAT_INTEL.md`. |

3.  **Cross-Verification**: Use the `ThreatFeedClient` unit tests to verify payloads generated by the new CLI tool.
