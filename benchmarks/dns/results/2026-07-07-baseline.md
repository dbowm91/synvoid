## How To Use This Baseline

This file is a **local reference**, not a CI pass/fail gate. To compare future runs:

1. Re-run `./scripts/dns/run_benchmarks.sh --all` on the same hardware.
2. Diff the criterion `time: [...]` lines against this file.
3. Treat any single-benchmark regression > 10% as worth investigating, not a hard fail. Criterion reports confidence intervals; cross-check before declaring a regression.
4. Do **not** compare across machines. Cross-platform noise dominates.

If the bench harness changes (new benchmarks added, criterion options changed), create a new dated baseline file rather than overwriting this one.

## Notes

- All benchmarks run with `cargo bench -p synvoid-dns` in release profile.
- Criterion reports min / mean / max over 100 samples per benchmark (3s warmup).
- Outlier counts are reported as a sanity-check for noise; they are not used as pass/fail.
- A single-benchmark regression > 10% on the same hardware should be reviewed, not blindly accepted as a regression.
- This baseline is **manual / reference only**. CI does not run benchmarks and does not enforce timings.
